Natural Language Processing


While the sub-reddit dataset contains different types of data, the most important information remains in the text. Thus, we combine and clean all the meaningful text and apply different NLP pipelines to the text column to finally extract the important information, specifically, features in the timeline.

We place our main focus on the sentiment analysis and topic modeling. While sentiment analysis gives us the daily average sentiment scores, topic modeling helps us extract the most significant topics and return the daily topic proportions, which are used as features in the later modeling for stationary patterns in the timeline.


Sentiment Analysis

We make use of the pre-trained pipeline of analyze_sentimentdl_use_twitter provided by John Snow, and apply it to both the submission and comment datasets. Then we assign the sentiment scores of 'positive' reddits being 1.0 and others being 0.0. Finally we aggregate the sentiment scores by averaging them on a daily basis.

We plot the final daily sentiment scores as well as the moving-average trend on a 10-day basis for both the submission and comment datasets.

Topic Modeling

We use the LDA model provided in SparkMLlib. We contruct the whole pipeline using DocumentAssembler, Tokenizer, stopwords_cleaner_v2, finisher, CountVectorizer and finally LDA.

We proceed with 15 topics, top 10 words per topic using this pipeline, for both the submission and comment datasets.

After we tag the reddits using the topic words and calculate daily proportions, finally we plot the daily topic proportions in the timeline.

NLP Features

Finally we have the NLP features of sentiment scores and topic proportions in the timeline, for both submissions and comments.

We expect to build models with these features and targets as daily returns of specific commodities in the external dataset, in order to explore how well the R-squared scores we can achieve from the models and whether some interesting findings exist.
